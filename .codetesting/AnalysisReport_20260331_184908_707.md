### Analysis
The production code at JudgeRegistrationRazor/Pages/Index.cshtml.cs line 27 uses `Config?["HomePage"]` to access a dictionary value. The null-conditional operator (`?.`) only protects against Config being null, but does not protect against the "HomePage" key being missing from the dictionary. When the HomePage key doesn't exist in the Config dictionary (which is a valid scenario as demonstrated by the test), a KeyNotFoundException is thrown. The code should use TryGetValue or check ContainsKey before accessing the dictionary value to handle cases where the HomePage configuration is not present in the database.

### Suggested Fix
Replace `return Redirect(Config?["HomePage"]);` with safe dictionary access, such as: `return Config?.TryGetValue("HomePage", out var homePage) == true ? Redirect(homePage) : Redirect("/");` or check with `ContainsKey` before accessing. Alternatively, use `GetValueOrDefault()` with a fallback URL.
